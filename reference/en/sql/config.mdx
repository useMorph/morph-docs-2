title: 'config'
---

You can write configuration values in SQL using jinja as follows.

```sql
{{
	config(
		name="test",
		description="this is test sql statement",
		output_paths=[
			"/tmp/morph/{name}/{now()}{ext()}"
		],
		connection="your_connection_slug",
		result_cache_ttl=0
	)
}}

select 1 as test
```

## Parameters

<ParamField body="name" type="string" default="SQL file name">
The process name, which can also be used as an alias when called from other files.
</ParamField>

<ParamField body="description" type="string">
You can describe the process.
</ParamField>

<ParamField body="output_paths" type="array: string" default="['/tmp/morph/{name}/{now()}{ext()}']">
The output destination of the result file.

Refer to [here](/reference/en/framework/output-paths) for the description method.
</ParamField>

<ParamField body="connection" type="string" default="default_connection set in morph_project.yml">

</ParamField>

Specify the cache expiration time of the data pipeline execution result in seconds.
</ParamField>
