# Problem

以下のようにMDXのプレビューやPagesのセクションを開いた時に「Bad Gateway」と表示されるようになってしまった場合の解決策について解説します。

    alt="MDX Bad Gateway"

# Solution

原因としては、MDXのプレビューを表示するためのViteサーバーが起動していないことにあります。
この場合、以下の2通りの方法で解決をすることができます。

## 解決策1: 画面全体をリロードする

Morphのワークスペースでは、起動された時に自動的にViteサーバーを起動しています。
そのため、通常であれば画面をリロードし直すことによってViteサーバーが再起動されて画面が正常に表示されるようになります。

## 解決策2: ターミナルからViteサーバーを起動する

Codeセクションからターミナルに直接以下のコマンドを実行してください。

このコマンドが正常に起動する場合は、Viteサーバーの起動が終わり次第プレビューにアクセスすることが可能になります。

```shell
npm run 'page:start'
```

## それでも解決しない場合

上記の手段でも解決しない場合は、パッケージの初期化に失敗している可能性があります。

[こちら](/data-application/ja/mdx-setup)のページに記載されているMDXのセットアップコマンドをターミナルで実行しコマンドの初期化を行なった後に上記の解決策を実行してください。
